## The Problem

Every mutual fund cost calculator stops at the expense ratio. That number covers management fees, 12b-1 fees, and administrative expenses. It does not cover brokerage commissions paid when the fund trades, the bid-ask spreads on those trades, market impact from large orders, soft dollar arrangements where the fund pays inflated commissions in exchange for "free" research, revenue sharing agreements where fund companies pay brokers for placement on recommended lists, or the tax drag generated by excessive portfolio turnover.

We call these pre-NAV trading costs **sub-NAV drag** — the costs that hit a fund's returns before the NAV is even calculated, before the expense ratio is deducted, and long before you see your statement.

The data to quantify all of this has been sitting in SEC filings since 2018. Machine-readable, publicly available, filed by every fund in America. Nobody built a tool to stitch it together.

Fund Autopsy does.

## Quick Start

```bash
pip install fundautopsy[web]
python -m fundautopsy.web
# Open http://localhost:8000
```

Type any mutual fund ticker. Fund Autopsy pulls live data from SEC EDGAR and shows you what your fund actually costs.

## Data Sources

Fund Autopsy currently parses three filing types, with more in development:

| Filing | What It Provides | Frequency |
|--------|-----------------|-----------|
| **N-CEN** | Brokerage commissions, soft dollar arrangements, affiliated broker usage, securities lending revenue, service providers | Annual |
| **N-PORT** | Complete portfolio holdings, asset class breakdown, net assets, liquidity classification | Quarterly |
| **497K** | Expense ratio, management fee, 12b-1 fee, other expenses, turnover rate | Per share class |

All data retrieved live from SEC EDGAR. No proprietary data, no scraping, no third-party APIs.

## How It Works

1. **Resolve** — Ticker maps to CIK, series ID, and class ID via SEC's mutual fund ticker file
2. **Retrieve** — Pull the most recent N-CEN, N-PORT, and 497K filings for the matched series
3. **Compute** — Brokerage commissions from N-CEN (actual dollars), bid-ask spreads and market impact estimated from N-PORT asset mix and turnover rate
4. **Present** — Stated expense ratio alongside true total cost range, with every component tagged by source filing

Multi-series trusts (Fidelity, Vanguard, etc.) are handled by iterating through filings to match the correct series rather than assuming the most recent filing belongs to your fund. Custom HTML parsers handle the three most common 497K table formats across fund families.

## The Sub-NAV Drag Framework

Traditional cost analysis looks like this: **Gross return - Expense ratio = Net return**

The real picture:

```
Hypothetical gross return (if the fund could trade for free)
  minus  Sub-NAV drag (commissions, spreads, market impact, soft dollars)
= Reported gross return (what the fund calculates NAV from)
  minus  Expense ratio (management fee + 12b-1 + admin)
= Net return (what you actually get)
```

Sub-NAV drag is not new — the academic literature has studied trading costs for decades. What's new is naming it as an aggregate fund-level metric, quantifying it from structured SEC filings, and making it accessible to anyone with a ticker symbol.

See [docs/methodology.md](docs/methodology.md) for the full framework with academic citations.

## Known Limitations

1. N-CEN data starts in 2018. No structured soft dollar data before that.
2. N-CEN is filed annually. Commission data may be up to 12 months old.
3. Bid-ask spread and market impact are model estimates, not observed execution costs.
4. SAI, N-CSR, Form CRS, and Form ADV parsers are not yet implemented. Current analysis is limited to N-CEN, N-PORT, and 497K data.
5. Tax drag is excluded from current scope.
6. Fund-of-funds (target-date funds) do not recursively unwind underlying fund costs.
